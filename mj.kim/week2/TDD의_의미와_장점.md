# TDD의 의미와 장점

## Test-Driven Development

> 테스트 주도 개발이라고도 하며, 소프트웨어 개발 방법론 중 하나로, 테스트 코드를 먼저 작성하고 이를 통과시키는 코드를 개발하는 방법
> 소프트웨어 품질을 높이는데 많은 도움을 주는 방법론 중 하나이며 특히, 큰 규모의 프로젝트나 복잡한 로직을 다룰 때 효과적

### TDD의 단계

1. 테스트 코드 작성

- 먼저, 개발할 기능에 대한 요구사항을 정리
- 이후, 해당 기능을 검증하는 테스트 코드 작성

2. 테스트 통과를 위한 코드 작성

- 처음에는 작성된 코드를 실행해보면 해당 테스트가 실패함
- 이후, 하나하나씩 요구사항에 맞춰 개발하면서 테스트를 통과시키는 코드를 작성

3. 리팩토링

- 작성한 코드에 대해 리팩토링을 수행
- 이 단계에서는 코드의 가독성, 유지보수성, 성능 등을 개선

### TDD 장점

- 먼저 테스트 코드를 작성하므로 코드의 기능을 명확하게 이해하게 됨
  - 이를 바탕으로 실제 개발에 들어가서 적절한 코드를 작성할 수 있음
- 테스트 코드를 작성함으로써 요구사항에 대한 버그를 미리 발견할 수 있음
- 코드의 변경에 따른 영향도를 파악할 수 있음
- 코드의 리팩토링을 통해 코드의 품질을 높일 수 있음

### TDD의 단점

- 생산성 저하
  - 처음에 모든 요구사항 기능에 대한 테스트 코드를 만들고 시작해야 되기 때문에 `테스트코드`, `실제코드` 두 개를 짜야되기 때문에 개발 속도가 느려질 수 밖에 없음
    위와 같은 치명적 단점때문에 SI프로젝트 딴에서는 소프트웨어의 품질보다 납기일 준수가 훨씬 중요하기 때문에 TDD방식을 잘 사용하지 않음
